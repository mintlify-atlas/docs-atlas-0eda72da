## Prerequisites

Before you begin, ensure you have the following installed:

- **Java Development Kit (JDK) 19+**
- **Apache Maven 3.6+**
- **Git** for version control
- **IDE** (NetBeans, IntelliJ IDEA, or Eclipse)

## Clone the Repository

```bash
git clone <repository-url>
cd patientManager
```

## Building from Source

### Compile the Project

```bash
mvn clean compile
```

### Run Tests

```bash
mvn test
```

### Package as JAR

```bash
mvn clean package
```

This creates an executable JAR in `target/patientManager-1.0-SNAPSHOT.jar`.

### Run the Application

```bash
mvn exec:java
```

```bash
java -jar target/patientManager-1.0-SNAPSHOT.jar
```

## Development Database

The application uses SQLite with a user-specific database location.

### Database Location

The database is automatically created at:

```
~/.patientManager/patient.db
```

On Windows:
```
C:\Users\<username>\.patientManager\patient.db
```

On Linux/Mac:
```
/home/<username>/.patientManager/patient.db
```

### Database Initialization

The `JPAUtil` class handles database initialization (from `JPAUtil.java:26-54`):

```java
Path appData = Paths.get(System.getProperty("user.home"), ".patientManager");
Path dbFile = appData.resolve("patient.db");

try {
    if (!Files.exists(appData)) {
        Files.createDirectories(appData);
    }

    // Copy db from JAR if not exists
    if (!Files.exists(dbFile)) {
        try (InputStream is = JPAUtil.class.getResourceAsStream("/data/patient.db")) {
            if (is == null) {
                throw new FileNotFoundException("Database template not found");
            }
            Files.copy(is, dbFile);
        }
    }
} catch (IOException e) {
    throw new RuntimeException("Error preparing database", e);
}
```

### Resetting the Database

To reset your development database:

```bash
# Delete the existing database
rm ~/.patientManager/patient.db

# Run the application (it will recreate from template)
mvn exec:java
```

## Code Style Guidelines

### Naming Conventions

- **Classes**: PascalCase (e.g., `PatientService`, `BaseRepository`)
- **Methods**: camelCase (e.g., `findById`, `createPatient`)
- **Variables**: camelCase (e.g., `patientId`, `entityManager`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `MAX_RETRY_COUNT`)

### Package Organization

- `model` - JPA entity classes
- `repository` - Data access objects (DAOs)
- `service` - Business logic
- `gui` - Swing UI components
- `gui.component` - Custom reusable UI components
- `gui.theme` - Theme management
- `utils` - Utility classes

### Code Documentation

Use JavaDoc for public APIs:

```java
/**
 * Service class for managing patient operations.
 * Provides CRUD operations and business logic for patients.
 */
public class PatientService {
    
    /**
     * Creates a new patient in the database.
     * 
     * @param patient The patient entity to create
     * @return The persisted patient with generated ID
     */
    public Patient create(Patient patient) {
        return patientRepository.create(patient);
    }
}
```

## Troubleshooting

### Maven Build Fails

**Problem**: Dependencies cannot be downloaded

**Solution**:
```bash
mvn clean install -U
```

### Database Lock Errors

**Problem**: "Database is locked" error

**Solution**:
- Close all other instances of the application
- Check for zombie processes: `ps aux | grep java`
- Delete lock files in `~/.patientManager/`

### UI Rendering Issues

**Problem**: Components not displaying correctly

**Solution**:
- Ensure FlatLaf is properly initialized
- Check theme resources are accessible
- Verify Java version compatibility (requires JDK 19+)

### JPA Configuration Errors

**Problem**: EntityManager creation fails

**Solution**:
- Verify `persistence.xml` is in `src/main/resources/META-INF/`
- Check database file permissions
- Review Hibernate logs for detailed errors
