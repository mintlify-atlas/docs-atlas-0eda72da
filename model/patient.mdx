## Overview

The `Patient` entity is the central entity in the Patient Manager application, representing an individual receiving psychological services. It stores comprehensive personal information, contact details, and treatment status.

**Source:** `src/main/java/com/bo/patientmanager/model/Patient.java`

## Entity Definition

```java
@Entity
public class Patient {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long patientId;
    
    // Personal information
    private String name;
    private String lastName;
    private Date birthday;
    private String gender;
    private String avatar;
    
    // Contact information
    private String address;
    private String city;
    private String country;
    private String phone1;
    private String phone2;
    private String email;
    private String social;
    
    // Treatment information
    private String derivedFrom;
    private String derivedTo;
    @Column(length = 4000)
    private String observations;
    
    // Status
    private boolean isCurrentPatient;
    private State state;
}
```

## Patient States

The `State` enum defines four possible treatment states:

<AccordionGroup>
  <Accordion title="ACTIVO - Active Treatment">
    Patient is currently receiving active treatment and attending sessions.
    
    **Display Value:** "Activo"
    
    This is the default state for new patients.
  </Accordion>

  <Accordion title="ALTA - Discharged">
    Patient has successfully completed treatment and been discharged.
    
    **Display Value:** "Alta"
    
    Used when treatment goals have been achieved.
  </Accordion>

  <Accordion title="ABANDONO - Abandoned">
    Patient has discontinued treatment without proper discharge.
    
    **Display Value:** "Abandono"
    
    Used when patient stops attending sessions without completion.
  </Accordion>

  <Accordion title="DERIVACION - Referred Out">
    Patient has been referred to another provider or specialist.
    
    **Display Value:** "Derivación"
    
    Used when patient needs services outside the practice's scope.
  </Accordion>
</AccordionGroup>

### State Enum Definition

```java
enum State {
    ACTIVO,      // Active
    ALTA,        // Discharged
    ABANDONO,    // Abandoned
    DERIVACION   // Referred
}
```

### State Display Logic

The `getState()` method converts enum values to display strings:

```java
public String getState() {
    return switch (this.state) {
        case ACTIVO    -> "Activo";
        case ALTA      -> "Alta";
        case ABANDONO  -> "Abandono";
        case DERIVACION -> "Derivación";
        default -> null;
    };
}
```

## Default Values

When creating a new Patient instance, the constructor sets default values:

```java
public Patient() {
    this.isCurrentPatient = true;
    this.state = State.ACTIVO;
}
```

This ensures that:
- New patients are automatically marked as current
- New patients start in the "ACTIVO" (active) state

## Relationships

### Sessions

Patients can have multiple scheduled therapy sessions:

```java
// In Session.java
@ManyToOne
@JoinColumn(name = "patient_id", nullable = false)
private Patient patient;
```

Each session includes:
- Day of the week
- Start and end times
- Session mode (in-person, virtual, etc.)
- Billing information

See [Session Entity](/model/session) for details.

### Relatives

Patients can have multiple associated family members and contacts:

```java
// In Relative.java
@ManyToOne
@JoinColumn(name = "patient_id", nullable = false)
private Patient patient;
```

See [Relative Entity](/model/relative) for details.

### Patient-Relative Relations

The many-to-many relationship between patients and relatives is managed through `PatientRelativeRelation`, which tracks:
- Specific relationship type
- Whether relative is main contact
- Observations about the relationship

## Usage Examples

### Creating a New Patient

```java
Patient patient = new Patient(
    "Juan",                    // name
    "Pérez",                   // lastName
    new Date(),                // birthday
    "Male",                    // gender
    "/avatars/default.png",   // avatar
    "Calle Principal 123",     // address
    "Buenos Aires",            // city
    "Argentina",               // country
    "+54 11 1234-5678",        // phone1
    "+54 11 8765-4321",        // phone2
    "juan@example.com",        // email
    "20-12345678-9",          // social
    "Dr. García",             // derivedFrom
    "Initial consultation notes" // initialObservations
);
```

### Updating Patient State

```java
// Discharge a patient
patient.setState(State.ALTA);

// Mark patient as referred out
patient.setState(State.DERIVACION);
patient.setDerivedTo("Dr. Specialist Name");

// Mark as abandoned
patient.setState(State.ABANDONO);
patient.setIsCurrentPatient(false);
```

### Querying Current Patients

```java
// Find all active patients
List<Patient> activePatients = patientRepository.findByIsCurrentPatientTrue();

// Find patients by state
List<Patient> discharged = patientRepository.findByState(State.ALTA);
```

## Data Validation

<Warning>
  The Patient entity does not include built-in validation annotations. Implement validation in the service layer or DTOs:
  - Validate email format
  - Ensure required fields (name, lastName) are not empty
  - Validate date ranges for birthday
  - Validate phone number formats
</Warning>
