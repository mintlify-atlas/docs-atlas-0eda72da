## Overview

The repository layer provides data access functionality for all entities in the Patient Manager application. All repositories extend `BaseRepository`, which provides common transaction management and JPA EntityManager handling.

## Architecture

### Inheritance Hierarchy

```
BaseRepository (abstract)
├── PatientRepository
├── ClinicalHistoryRepository
├── SessionRepository
├── RelativeRepository
├── PatientRelativeRelationRepository
├── RelativeObservationRepository
└── ClinicalHistoryFileRepository
```

### Key Features

- **Transaction Management**: Automatic transaction begin, commit, and rollback
- **Resource Management**: Automatic EntityManager lifecycle handling
- **Error Handling**: Automatic rollback on exceptions
- **Reusable Patterns**: Common CRUD operations inherited from BaseRepository

#### execute

```java
protected <T> T execute(Function<EntityManager, T> action)
```

Executes a database operation within a transaction with automatic resource management.

**Parameters**:
- `action` - A function that performs operations using the EntityManager

**Returns**: The result of the action function

**Behavior**:
1. Creates a new EntityManager
2. Begins a transaction
3. Executes the provided action
4. Commits the transaction on success
5. Rolls back the transaction on error
6. Closes the EntityManager in all cases

**Example**:
```java
public Patient create(Patient patient) {
    return execute(em -> {
        em.persist(patient);
        return patient;
    });
}
```

**Location**: `BaseRepository.java:28`

## Usage Patterns

### Creating a Repository Instance

All repositories require an EntityManagerFactory during initialization:

```java
import jakarta.persistence.EntityManagerFactory;
import persistence.JPAUtil;

EntityManagerFactory emf = JPAUtil.getEntityManagerFactory();
PatientRepository patientRepo = new PatientRepository(emf);
```

### Basic CRUD Operations

```java
// Create
Patient patient = new Patient();
patient.setName("John Doe");
patient.setEmail("john@example.com");
patient = patientRepo.create(patient);

// Read
Patient found = patientRepo.findById(patient.getId());

// Update
found.setEmail("newemail@example.com");
found = patientRepo.update(found);

// Delete
patientRepo.delete(patient.getId());
```

### Querying with Relationships

```java
// Get all sessions for a patient
SessionRepository sessionRepo = new SessionRepository(emf);
List<Session> sessions = sessionRepo.findByPatientId(patientId);

// Count patient sessions
Long count = sessionRepo.countByPatientId(patientId);

// Get patient relatives
PatientRelativeRelationRepository relationRepo = 
    new PatientRelativeRelationRepository(emf);
List<PatientRelativeRelation> relatives = 
    relationRepo.findByPatientId(patientId);

// Get observations for a relative relationship
RelativeObservationRepository obsRepo = 
    new RelativeObservationRepository(emf);
List<RelativeObservation> observations = 
    obsRepo.findByRelation(relationId);
```

### Transaction Management

Transactions are automatically managed by the `execute()` method in BaseRepository:

```java
// This operation runs in a transaction automatically
Patient patient = patientRepo.create(new Patient());

// If an exception occurs, the transaction is rolled back
try {
    patient.setEmail("invalid-email");
    patientRepo.update(patient); // May throw validation exception
} catch (Exception e) {
    // Transaction already rolled back
    System.err.println("Update failed: " + e.getMessage());
}
```

### Error Handling

All repository methods will throw RuntimeException if a database error occurs. The transaction is automatically rolled back:

```java
try {
    Patient patient = patientRepo.findById(999L);
    if (patient == null) {
        throw new IllegalArgumentException("Patient not found");
    }
    // Process patient...
} catch (RuntimeException e) {
    // Handle database errors
    logger.error("Database operation failed", e);
}
```

## Best Practices

### 1. Repository Initialization

Initialize repositories once and reuse them:

```java
// Good: Initialize once
EntityManagerFactory emf = JPAUtil.getEntityManagerFactory();
PatientRepository patientRepo = new PatientRepository(emf);

// Use multiple times
Patient p1 = patientRepo.findById(1L);
Patient p2 = patientRepo.findById(2L);
```

### 2. Resource Management

EntityManager lifecycle is handled automatically - no manual closing needed:

```java
// BaseRepository handles this automatically
Patient patient = patientRepo.findById(id);
// EntityManager is closed after the operation
```

### 3. Transaction Boundaries

Each repository method runs in its own transaction:

```java
// Each method is a separate transaction
Patient patient = patientRepo.create(new Patient());
Session session = sessionRepo.create(new Session());

// If you need atomic operations, use a service layer
```

### 4. Query Optimization

Use specific query methods instead of loading all entities:

```java
// Good: Specific query
List<Session> sessions = sessionRepo.findByPatientId(patientId);

// Avoid: Loading all and filtering
List<Session> allSessions = sessionRepo.findAll();
// Then filter in Java...
```

### 5. Null Checking

Always check for null when using findById:

```java
Patient patient = patientRepo.findById(id);
if (patient == null) {
    throw new IllegalArgumentException("Patient not found with ID: " + id);
}
```
